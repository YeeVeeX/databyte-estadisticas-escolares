Validación de Contraseña

def validar_contrasena(clave):
    checklist = {
        "largo": len(clave) >= 8,
        "mayus": False, "minus": False,
        "numero": False, "simbolo": False,
    }
    for letra in clave:
        if letra.isupper(): checklist["mayus"] = True
        elif letra.islower(): checklist["minus"] = True
        elif letra.isdigit(): checklist["numero"] = True
        elif letra in "!@#$%^&*()": checklist["simbolo"] = True
Diseño: Iteramos carácter por carácter con métodos nativos de string (isupper, isdigit), sin expresiones regulares.

Apropiado para el nivel del curso y fácil de explicar en la presentación.

Obtener Edad Mayor

def obtener_edad_mayor(alumnos):
    mayor = -1
    nombre_mayor = ""
    for nombre in alumnos:
        if alumnos[nombre] > mayor:
            mayor = alumnos[nombre]
            nombre_mayor = nombre
    return nombre_mayor, mayor
Concepto clave: En lugar de usar max() built-in, implementamos el algoritmo manualmente.

Demuestra comprensión de cómo funciona internamente y devuelve dos datos (nombre y edad) al mismo tiempo con return múltiple.

Cálculo del Promedio

def calcular_promedio(alumnos):
    suma = 0
    cantidad = 0
    for nombre in alumnos:
        suma += alumnos[nombre]
        cantidad += 1
    promedio = suma / cantidad
    return round(promedio, 1)
¿Por qué round(promedio, 1)? El cliente pidió explícitamente "un decimal".
¿Por qué no sum() ni len()? Implementar manualmente enseña el patrón de acumulador, fundamental en programación.

Cálculo de la Mediana

def calcular_mediana(alumnos):
    edades = []
    for nombre in alumnos:
        edades.append(alumnos[nombre])
    edades.sort()

    cantidad = len(edades)
    mitad = cantidad // 2

    if cantidad % 2 == 1:
        return edades[mitad]             # Caso impar: valor central
    else:
        return (edades[mitad - 1] + edades[mitad]) / 2  # Caso par

El truco: cantidad % 2 == 1 detecta cantidad impar. En ese caso tomamos el elemento central directo. En caso par, promediamos los dos centrales.

Ejemplo: Mediana en Acción

Muestra impar (n=5)

[7, 8, 9, 10, 12]

mitad = 5 // 2 = 2

edades[2] = 9

Mediana: 9
Muestra par (n=4)

[7, 8, 10, 12]

mitad = 4 // 2 = 2

(8 + 10) / 2 = 9.0

Mediana: 9.0

La diferencia entre 9 (entero) y 9.0 (flotante) es correcta — Python infiere tipos según la operación de división.

frames = {}

def mostrar_frame(nombre):
    for key in frames:
        frames[key].pack_forget()
    frames[nombre].pack(fill="both", expand=True)

Reloj en Vivo

def actualizar_reloj():
    ahora = datetime.now()
    reloj_var.set(ahora.strftime("%H:%M:%S"))
    fecha_var.set(ahora.strftime("%A, %d de %B del %Y"))
    ventana.after(1000, actualizar_reloj)
Concepto clave: ventana.after(1000, funcion) programa la ejecución en 1 segundo sin bloquear la interfaz.
Nunca usar time.sleep() en GUI — congelaría la pantalla.

def actualizar_checklist(evento=None):

    for clave_c in checklist:

Seguridad: Intentos Limitados

def intentar_login():
    global intentos_restantes
    ingresado = entry_pwd.get()

    if ingresado == clave_guardada:
        mostrar_frame("muestra")
        return

    intentos_restantes -= 1
    if intentos_restantes <= 0:

Práctica de seguridad real: 3 intentos máximo. Al llegar a 0, la aplicación se cierra.

Simula el comportamiento de sistemas bancarios y empresariales.

¿Por qué HTML y no PDF?

  • Cero librerías — reportlab/fpdf requieren pip
  • Look profesional con CSS (gradientes, cards, tablas)
  • Ctrl+P → "Guardar como PDF" desde el navegador
  • Universal — cualquier sistema lo abre

También generamos un TXT plano como respaldo compatible.

Guardado con Timestamp

def guardar_reporte(contenido, extension):
    ahora = datetime.now()
    nombre = "reporte_" + ahora.strftime("%Y-%m-%d_%H-%M-%S") + "." + extension
    ruta = os.path.join("reportes", nombre)

    archivo = open(ruta, "w", encoding="utf-8")
    archivo.write(contenido)
    archivo.close()
    return ruta
Cada reporte tiene nombre único basado en fecha/hora. Nunca se sobrescriben — todos los reportes históricos se conservan automáticamente.

¿Por qué HTML?

Lecciones Aprendidas

Técnicas

  • Separar lógica de interfaz permite reutilizar código
  • tkinter puede verse moderno con el tema y paleta correctos
  • after() es la forma correcta de tareas periódicas en GUI
  • bind("<Return>", ...) mejora la UX sin complejidad extra

Conceptuales

  • Los diccionarios son ideales para relacionar datos (nombre → edad)
  • La mediana requiere ordenar primero y decidir par/impar
  • Validaciones en tiempo real mejoran la experiencia del usuario
  • HTML + CSS da mejor resultado visual que muchas librerías PDF
